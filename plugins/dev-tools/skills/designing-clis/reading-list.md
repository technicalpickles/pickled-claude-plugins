# CLI Design Reading List

## Foundational UX Theory

| Resource | Type | Why It Matters for CLIs |
|----------|------|------------------------|
| **Don Norman - The Design of Everyday Things** | Book | Affordances, feedback, error recovery - core to making CLIs that communicate clearly |
| **Steve Krug - Don't Make Me Think** | Book | Usability and simplicity philosophy - predictable commands, minimal friction |
| **Jakob Nielsen - 10 Usability Heuristics** | Article | Visibility of system status, consistency, error prevention - explains why feedback builds confidence |
| **Alan Cooper - About Face** | Book | Goal-oriented design - create tools that feel supportive and responsive |

## Developer Experience & Terminal UX

| Resource | Type | Why It Matters for CLIs |
|----------|------|------------------------|
| **Charm.sh - Designing Command-Line Tools That Feel Good** | Blog | Thoughtful layout, color, copy - shows how details create delight |
| **Warp Blog - The Future of the Command Line** | Blog | Reimagining terminal with structure and feedback - GUI-level UX in text |
| **GitHub Blog - Bringing GitHub to the Terminal** | Blog | Mirrors Git syntax while adding guided prompts - model for discoverability |
| **Scott Jenson - The UX of Command Lines** | Essay | Command lines can be elegant and discoverable with intentional design |
| **Tom MacWright - Interface Without Interface** | Blog | Minimalist philosophy - know when to remove, not add |

## Visual Design & Terminal Aesthetics

| Resource | Type | Why It Matters for CLIs |
|----------|------|------------------------|
| **Edward Tufte - The Visual Display of Quantitative Information** | Book | Signal-to-noise ratio, visual hierarchy - clarity in dense output |
| **Lidwell, Holden, Butler - Universal Principles of Design** | Book | Proximity, alignment, visibility - organizing terminal output |
| **Command Line Interface Guidelines (clig.dev)** | Guide | Community best practices for conventions, flags, help, language |
| **Atlassian - 10 Design Principles for Delightful CLIs** | Blog | Visual feedback, guidance, error messaging examples |

## Tools & Libraries

| Tool | Language | Purpose |
|------|----------|---------|
| **Rich** | Python | Beautiful CLI output with tables, markdown, live widgets |
| **Chalk** | Node.js | Color and style for highlighting errors, success, labels |
| **Blessed** | Node.js | Full TUI framework with panels, inputs, keybindings |
| **Bubbletea** | Go | Framework for building terminal UIs with modern patterns |

## Advanced Thinking

| Resource | Type | Why It Matters for CLIs |
|----------|------|------------------------|
| **Bret Victor - Learnable Programming** | Essay | Feedback loops and interactive explanations - making CLIs educational |
| **Jef Raskin - The Humane Interface** | Book | Psychology of flow, error prevention, predictable feedback |
| **Intercom - Intercom on Product Design** | Essays | Tone, guidance, clarity - writing help text that feels human |

## How to Use This List

**Building your first CLI?** Start with:
- Command Line Interface Guidelines (clig.dev)
- Charm.sh blog post
- GitHub CLI blog post

**Improving UX?** Read:
- Don Norman (The Design of Everyday Things)
- Steve Krug (Don't Make Me Think)
- Jakob Nielsen heuristics

**Working on visual design?** Study:
- Edward Tufte (data display principles)
- Universal Principles of Design
- Terminal aesthetics resources

**Need implementation help?** Explore:
- Rich (Python)
- Chalk (Node.js)
- Blessed (Node.js TUIs)
